# ステータス/ワークフロー/トラッカー

## 概要

- 課題はトラッカー（種別）とステータスを持ち、遷移・編集可否はワークフロー設定で制御される。
- ワークフローは一般に「(ロール × トラッカー) ごとの遷移」「(ロール × トラッカー × ステータス) ごとのフィールド権限/必須」等で表現される。

## 関連する主要モデル（概念）

- IssueStatus: ステータスマスタ
- Tracker: トラッカー（課題種別）
- WorkflowTransition / WorkflowRule / WorkflowPermission: 遷移とフィールド制御の表現（実体の名称は実装に依存）

## 代表的な仕様観点

- 遷移可能なステータス: 現在ステータス/ロール/トラッカー/プロジェクト設定に依存
- ステータス変更の権限: 変更可否（および操作UIの表示）は権限チェックとワークフロー制御の合成
- 必須フィールド/閲覧不可フィールド: ロール/ステータス等により変化し得る

## 管理画面

- 管理者がワークフロー編集画面から遷移/権限を設定できる前提（DB側設定）
