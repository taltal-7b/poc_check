# セキュリティ

## 認証

- セッションベースの認証に加え、APIキー/HTTP Basic 等の経路が存在する。
- 二要素認証が利用可能で、特定条件で有効化を促す挙動がある。

## CSRF

- API リクエスト相当の判定がある場合、CSRF トークン検証が別経路となる（API利用前提）。

## パスワードポリシー

- 最小長、最大有効期限、必要文字種などの設定キーが存在する。

## APIキー/専用キー

- REST API: 有効/無効の設定キーが存在する。
- システムAPI/メール取り込み: 専用の有効化フラグとキーが存在する。

## 添付

- サイズ/拡張子の制限が設定キーとして存在する。

## AIアシスト（想定）

- 生成結果の閲覧権限は、元のタスク/コメント/工数の閲覧権限と同等に制御する
- 外部AIへ送信する場合は、個人情報・機密情報のマスキングとプロジェクト単位の禁止設定を提供する
- 生成結果のキャッシュ保持期間を設定し、削除要求に対応できる設計とする
